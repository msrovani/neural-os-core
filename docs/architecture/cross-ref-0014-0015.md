### Critical gap analysis

1. **USB philosophy is entirely absent** — The "no autorun, devices require intent" paradigm (arguably the most innovative idea in ADR-0014) does not appear in ADR-0015 at all. MVP treats keyboard as a standard input device.

2. **NPU remains a black box** — ADR-0015 just has a hook `init_npu()`. The deep analysis of CGRA, XDNA, overlays, command queues is not translated into any actionable sprint.

3. **CorePools algorithm simplified** — The sophisticated P/E-aware `assign_cores()` is replaced by a generic "wake APs and let MLP decide rings."

4. **MHI is MVP-scoped to Dram only** — The tiered allocation vision (Vram, Nvme, Hdd) is acknowledged but explicitly deferred. The expanded tier outputs (sfs_cold, tensor_swap, skill_heap) are not mentioned.

5. **Architectural principles dropped** — The "VirtIO first", "Polling > Interruption", "No HAL", "No hotplug" principles are not carried into ADR-0015.
